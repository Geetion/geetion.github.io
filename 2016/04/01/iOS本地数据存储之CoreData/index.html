iOS本地数据存储之CoreData

Core Data是苹果官方推荐的数据存储方式，相当于一个已经封装好了的数据库，如果学过sql的童鞋理解起来肯对会比较容易，说实话我是先接触的Core Data后来学PHP才开始了解Sql，所以一开始真的什么都记不住也不知道为什么，就是照猫画虎的用，所以现在回来重新梳理一下这个知识结构，顺便与SQL做对比方便后来人理解。
开启了Core Data的项目里面会多出来一个以xcdatamodeld为后缀的文件，这个文件就相当于Mysql里面的database数据库，需要用creat database XXX来创建在这里苹果已经为大家准备好了。
先说下MYSQL的命令行操作，需要先建立一个table在数据库里，也就是一个数据模型，所有的数据组都需要遵守这个模型，也就是Core Data文件里面的entities，想要存储一组数据就要先在这里定义好这组数据每个变量的类型和名称。
0.Mysql的数据库操作
在PHP里面操作Mysql数据库是需要先进行数据库连接的：

    header('Content-type:text/html;charset = utf-8');
    define('HOST', 'localhost');//设定地址
    define('USERNAME', 'root');//设定数据库用户名
    define('PASSWORD', '');//设定数据库密码

    $con = mysql_connect(HOST,USERNAME,PASSWORD)//连接数据库
    mysql_select_db('test') //选择数据库
而在Core Data里面则完全不需要管这些，因为一个项目也就这一个数据库，在Swift项目里面要使用的时候只需要获取一个上下文 
var context = (UIApplication.sharedApplication().delegate as! AppDelegate).managedObjectContext
为了方便管理数据我们可以让xcode给我们自动生成一个已经建好的entity的数据模型只需要在项目里面新建文件选择core data然后选择NSManagedObject Subclass，在这里如果是使用swift得项目的话需要建立一个bridge header文件导入这个oc的subclass ![3ADA15BB-C31F-4FC8-953C-FFC1ADD6EE95](http://www.ccjason.me/wp-content/uploads/2016/04/3ADA15BB-C31F-4FC8-953C-FFC1ADD6EE95-640x454.jpg) 1.插入数据 插入数据在mysql里面是需要语句 
"insert into tableName(title,author,description,content,dateline) values('title','author','description','content','dateline')"
也就是向名为tableName的table里面插入5个数据分别是title，author，description，content和dataline，这5个变量的值在后面value里面按顺序一一对应。 而在Core Data里面变成了这样 
let row = NSEntityDescription.insertNewObjectForEntityForName("tableName", inManagedObjectContext: self.context);//声明了一个entity模型用来插入名为TableName的entity
row.setValue(title, forKey: "title")；//分别向这个entity模型里面通过键值对的形式插入所需要的变量
row.setValue(description, forKey: "description")；
row.setValue(content, forKey: "content")；
row.setValue(dateline, forKey: "dateline")；
self.context.save()//让数据库存储改变
