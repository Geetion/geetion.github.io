BroadCast广播机制

为了方便系统级别的消息通知，安卓引入了一套广播消息机制。
广播主要分为两种：
1.标准广播：异步执行互不干涉
2.有序广播：进行完一个才能接着进行下一条广播

接收广播方式：
1.动态注册监听飞行模式
在Activity里面注册

建立一个类继承于BroadcastReciever，当接收到广播时会执行OnRecevie方法，重写onRecevie方法实现功能

2.静态注册监听飞行模式
在AndroidManifest文件中写好意图筛选

当然你也可以自定义广播，只需要把action里面的接收广播名填写自定义名称即可，这样做当然不会再有广播发过来了，那么你只有自己在需要的时候发送这条广播,例如我把action里面的名字改成了heheBroadCAST,然后在我需要的时候我只需要生成一个意图然后发送广播就可以了

想要发送有序广播和标准广播很类似，只需要把sendBroadCast改为
sendOrderedBroadcast();
这里会接收两个参数一个是参数接收器的句柄，另一个是权限相关字符，一般只需要传入Null就行
这样方式发送的标准广播不仅自己能接收到，如果别的程序也定义了相同的广播接收类型那么你发送的广播就会发送到别的应用程序，最关键的是你的应用程序有可能会接收到别的程序发来的消息，而很多消息可能是无用的，那么我们就需要使用本地通知了,也非常简单只需要获取到LocalBroadcastManager的单例

判断TextField为空引发的问题

一开始的判断方法是这样的在按确定键时执行，尝试了3种方法


    NSString *title = [_titleTextField text];

    if (title == nil){

    printf("222");

}

NSString *title = [_titleTextField text];

    if (title == @""){

    printf("222");

}

NSString *title = [_titleTextField text];

    if ([title isEqualToString: @""]){

    printf("222");

}


但是在不输入任何东西的情况下按确定依然没有执行这段代码，只有最后一种成功显示，以下是官方对textfield的text属性的注释


@property(nullable, nonatomic,copy)   NSString               *text;                 // default is nil

可以看出默认其实是为nil的首先要解决的是isEqualtoString于==的区别，而isEqualToString是isEqual的字符串扩展，所以就可以理解为isEqual于==的区别，具体参见http://blog.csdn.net/devday/article/details/7594669

工欲善其事必先利其器,亮剑Sublime

关于比较新的Atom和老牌的Sublime，一开是我其实挺喜欢Atom的，主要是因为界面，atom一脸的小清新，sublime一脸的土气，但是作为程序员当然不能只是看脸了，毕竟剑锋不锋利还得看刃而不是剑鞘，所以我百度整理了atom对于老牌sublime的优缺点：

  • 类似css语法的快捷键绑定，可编程性更高；门槛低；
  • 基于chrome，每个窗口都是web页，比起emacs更加容易定制界面，如果说emacs像操作系统，那么atom简直就是更现代的云操作系统，web页面能实现的形式，atom都可以。相比，sublime更专注编辑，很难通过自定义扩展搞这么花哨；
  • 整合nodejs，可以基于github上无数的package肩膀上，继续二次开发，资源多，成本小；
  • Tree视图等Atom更符合21世纪人的习惯
  • 源码集成上，在 Atom更改了代码如果没有提交，会在树视图上有颜色显示，作为源码版本控制github出品的Atom自然在这方面要强些，但是Sublime使用附加包就更强大了
  • 在性能上，Atom使用C++, Node.js, CoffeeScript构建，每个页面都是从本地HTML渲染，性能要差。
    来源：知乎等网站

总的来说atom就是界面好看，但是运行方面比sublime还是差一点，所以还是果断选择sublime

选择好了代码编辑器，那么插件当然必不可少了，要想快速撸代码全副武装sublime吧：

用package control 可以很方便的管理插件，而且安装简便，你只需要在sublime中按`ctrl+`` 调出console，然后前往package control 官网根据你的sublime版本复制右侧的一大堆代码到console中回车就安装好了。

插件首先是各种代码补全：

1.emmet：前身是大名鼎鼎的Zen coding，它使用仿CSS选择器的语法来生成代码，大大提高了HTML/CSS代码编写的速度

2.sublimerepl ：这可能是对程序员最有用的插件。SublimeREPL 允许你在 Sublime Texxt 中运行各种语言（NodeJS ， Python，Ruby， Scala 和 Haskell 等等）

Python常用库的记录

一.random模块：

1.random.randint(1,1000)用于随机选取一个数字，括号内为数字范围

2.list = [‘a’,’b’,’c’]

random.choice(list)用于从列表中选取一个元素

3.random.shuffle(list)用于把列表元素打乱顺序

二.sys模块

1.sys.exit()用于停止Python Shell程序或者控制台

2.sys.stdin.readline()读取用户输入的文本直到用户输入回车

3.sys.stdout()在某种程度上和print相同

3.sys.version显示当前Python版本信息

4.sys.execcutable显示当前文件目录信息

5.sys.getwindowsversion显示当先windows版本信息

6.sys.modules.keys()显示当前已导入的模块

 

三.time模块

1.time.time()用于返回1970年1月1日0点至今的描述

2.t = (2020,2,23,10,10,30,48,6,0,0)

该元组用于定义中环的时间参数上述参数分别为年,月,日,时,分,秒,星期几(此处从0开始计数，既0为星期一),一年中第几天,是否为夏令时间

time.asctime(t)用于转换日期

例如返回值：Sun FEB 23 10:30:48 2020

3.time.localtime()用于得到日期和时间

返回值例如：time.struct_time(tm_year = 2020, tm_mon = 2,tm_mday = 23,tm_hour = 22,tm_min = 18,tm_sec = 39,tm_wday = 0,tm_yday = 73,tm_isdst = 0)

t = time.localtime()

y = t[0]

print(y)

输出为：2020

4.time.sleep(5)使程序延迟括号内为秒数

四.pickle模块

data = {‘a’:’1’,’b’:’2’}

save_file = open(‘save.dat’,’wb’)(‘wb’以二进制模式写入文件)

pickle.dump(data,save_file)

save_file.close()

load_file = open(‘save.dat’,’rb’)

loaded_data = pickle(load_file)（反序列化）

load_file.close()

Objective-C 一个神奇的语言

其实一开始自认为已经可以熟练地使用swift编写一些程序了，再入这个OC的坑应该不会太难，但是事实证明：I’m wrrong…..

OC狠狠地把我调教了一顿，让我见识了什么是神一般的语言，我的第一感觉就是：尼玛，为什么要加星号啊，尼玛，为什么还有中括号，尼玛，为什么定义一个method还有加号和减号，为什么声明全局变量不能赋值。。。。请收下我的膝盖。。。。

好吧，不得不承认swift真的太方便了，但是OC看起来比较美观，因为同一段接口的官方文档给不懂代码的人看，大家一致认为OC的文档比较整洁，好吧我又给跪了。

前几天还看到有人在知乎上发帖问到底学OC好还是Swift好，其实自己一开始叶纠结过要学iOS开发应该学OC呢还是Swift呢，当然对于毛都不懂的一开始我来说，语法简介的Swift是有碾压型的优势的，不过现在反过头去看OC才发现其实OC虽然比较麻烦但是确实能学到很多东西，比如在Swift里面已经不用太多关注的Property等，还有OC的很多东西规范的很细致，比如implementation和interface，这两个东西在Swift里面已经合二为一了，再比如类方法和实例方法，好吧不得不说自己其实原来根本没有在意过这些东西，到了oc这真的学到了好多感觉用一辈子Swift都不一定懂的东西。

接下来整理一下这个过程中遇到过的问题：

1.为什么定义方法有加号和减号的区别：


-(NSString )executeSomeThing;
+(NSString )executeAnotherThing;

+表示该方法是类方法，不需要实例化该类就可以调用。
-表示该方法是实例方法，必须要实例化后才能调用

 

2.为什么要加星号，什么时候需要加星号：

OC的星号和C是一样的都是指针，如果属性的数据类型是基本类型，如本例中的age属性，属性前不用加上星号（当然，加上也不错，加上的话就表示是一个指针形的变量），否则如果数据类型是对象型的就必须加上星号

 

3.什么时候用中括号，什么时候用点：

个人经过实验认为调用方法的时候要用中括号，调用属性的时候要用点。

 

4.OC中属性的 定义：

格式：@property （特性） 属性类型 属性名称

assign：简单赋值，不更改索引计数

copy：建立一个索引数为1的对象，然后释放旧对象

retain：释放（release）旧对象，将旧对象的值赋予输入对象，再提高输入对象的索引计数为1

noatomatic： 非原子性访问，不加同步，多线程并发访问会提高性能。注意，如果不加此属性，
则默认是两个访问方法都为原子型事务访问。

strong:强引用（ARC）

weak：弱引用（ARC）
