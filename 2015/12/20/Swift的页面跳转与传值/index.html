Swift的页面跳转与传值

2015-12-20
#IOS #Swift #传值 #反向传值 #跳转
移动前端

虽然现在ios有了storyboard这个强大的可视化编程，但是个人还是喜欢用代码来编写页面跳转，首先说说用storyboard 的跳转和传值

segue，用的最多 的就是这个东西，也很简单只是要用右键拖动需要点按的按钮拖到另一个界面就完成了页面的跳转工作，这里在inspector里面有一个kind选项，有5种可选值，最常用的两个就是model和push，model是直接弹出页面，而push需要在第一个页面拥有navigationbar的情况下才能使用，不然运行会报错，如果要用这种方式传值的话就需要用代码来实现了首先需要给这个segue添加一个identifier，在第一个viewcotroller里面用prepareforsegue编写逻辑


//    segue页面跳转

override func prepareForSegue(segue: UIStoryboardSegue, sender: AnyObject?) {

/如果设置了很多segue的话就需要判断传进来得id是哪一个segue的以此来定位segue/

if segue.identifier == “rxNews” {

/指定目标界面的viewcontroller/

let vc = segue.destinationViewController as! WebViewController

/传值过去，现在这个VC就是目标界面的viewcontroller，id是目标界面viewcontroller中得变量*/

vc.id = “123”

}

}

下面再来说说用代码写界面跳转，顺便说说我遇到的几个坑：

还是先说push，直接上代码我一开始写的是这样的


                    /实例化webviewcontroller/

let push = WebViewController()

/传值/

push.id = “123123”

/界面跳转/

self.navigationController?.pushViewController(push, animated: true)

虽然我为viewcontroller绑定了storyboard但是传过去以后黑乎乎的一片，于是后来改成了这样

/实例化storyboard文件*/

var myStoryBoard = UIStoryboard(name: “Main”, bundle: nil)

/从storyboard里获取目标界面并绑定webviewcontroller/

let push = myStoryBoard.instantiateViewControllerWithIdentifier(“webview”) as! WebViewController

push.id = “123123”

self.navigationController?.pushViewController(push, animated: true)

区别在于，第一种viewcontroller是直接生成的并没有绑定storyboard（即使你已经在对应的viewcontroller绑定了storyboard因为在你进行实例化操作的时候第二个页面还没有生成所有也就没有完成绑定操作）而第二种方法是从storyboard里面提取到界面然后绑定(特别注意:用什么方式跳转viewcontroller就用跳转那个跳转的viewcontroller进行传值)

navigationbar还有很多页面跳转的方式，比如：

跳转到这个navigation的第一个界面


self.navigationController?.popToRootViewControllerAnimated(true)

返回到指定的viewcontroller（注意这里必须添这条navigationbar链里面出现过得viewcontroller，填别的运行报错）


self.navigationController?.popToViewController(XXXViewController, animated: true)

接下来是present其实和model 的效果一样


self.presentViewController(XXXViewController, animated: true, completion: nil)

和present专用的关闭回收界面的方式


self.dismissViewControllerAnimated(true, completion: nil)

 

接下来是反向传值，就是第二个界面传值到第一个界面，这里我又被上面遇到的问题坑了一次

协议传值

我本来是这样写的，第一个页面


/继承里面写协议名/

class ViewController: UIViewController,ModeViewControlDelegate {

@IBOutlet weak var lable: UILabel!

/实例化secondviewcontroller/

let second = SecondViewController()

 

override func viewDidLoad() {

super.viewDidLoad()

/设置secondviewcontroller的代理为本页面/

second.delegate = self

}

/实现代理的方法/

func changeprint(newString:String){

print(newString)

}

 

@IBAction func send(sender: AnyObject) {

self.navigationController?.pushViewController(second, animated: true)

}

}

第二个页面

/定义一个协议/

protocol ModeViewControlDelegate{

func changeprint(newString:String)

}

class SecondViewController: UIViewController {

/实例化协议/

var delegate:ModeViewControlDelegate?

override func viewDidLoad() {

super.viewDidLoad()

/传值/

delegate!.changeprint(“123”)

}

 

@IBAction func delegateButton(sender: AnyObject) {

self.navigationController?.popViewControllerAnimated(true)

}

}

大概流程就是先进入第一个viewcontroller点击按钮push到下一个view controller，第二个view controller生成完毕就传值给第一个view controller然后点击按钮返回第一个viewcontroller

这样下来我发现并没有什么卵用，值没有传过去，其实和我在跳转那里写到的原因是一样的，我实例化的viewcontroller和界面跳转的viewcontoller不是同一个

只要把 let second = SecondViewController()改成


let second = UIStoryboard(name: “Main”, bundle: nil).instantiateViewControllerWithIdentifier(“second”) as! SecondViewController就可以正常传值了

 

闭包传值


//      第二页面

class SecondViewController: UIViewController {

// 声明闭包

var changeTitleClosure:((Title:String)->Void)?

override func viewDidLoad() {

super.viewDidLoad()

// 发送数据

changeTitleClosure!(Title: “123”)

}

override func didReceiveMemoryWarning() {

super.didReceiveMemoryWarning()

}

@IBAction func closure(sender: AnyObject) {

self.navigationController?.popViewControllerAnimated(true)

}

}

// 第一个页面

class FirstViewController: UIViewController {

@IBOutlet weak var lable: UILabel!

// 生成Viewcontroller

let second = UIStoryboard(name: “Main”, bundle: nil).instantiateViewControllerWithIdentifier(“second”) as! SecondViewController

 

override func viewDidLoad() {

super.viewDidLoad()

// 写好接收的方法

second.changeTitleClosure = {(Title:String)->Void in

self.lable.text = Title

}

}

override func didReceiveMemoryWarning() {

super.didReceiveMemoryWarning()

}

@IBAction func send(sender: AnyObject) {

self.navigationController?.pushViewController(second, animated: true)

}

}
